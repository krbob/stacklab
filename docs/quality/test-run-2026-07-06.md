## Findingi (wg wagi)

### F-01 · Wysoka · Baseline write oznacza udany deploy jako FAILED
`internal/httpapi/handler.go:1796` (ścieżka create+deploy) oraz
`internal/maintenancejobs/service.go:248` (update wielu stacków). Po **udanym**
`up` (kontenery już działają) błąd `RecordDeployBaseline` powoduje
`markWorkflowFailed` + `FinishFailed("create_stack_failed")`. Użytkownik widzi
„deploy nieudany" mimo działającego stacka i może ponowić (→ konflikt). W update
wielostackowym przerywa pętlę — kolejne stacki nie są aktualizowane. Niespójne z
`handler.go:2856`, gdzie ten sam błąd jest tylko logowany (`Warn`). **Zalecenie:**
traktować błąd zapisu baseline jako `Warn`, nie jako porażkę deployu.

### F-02 · Średnia · Obejście `required` przez klucz ze spacjami
`internal/stacks/templates.go:337`. Druga pętla nadpisuje zwalidowane wartości po
`TrimSpace(key)`, więc `{" IMAGE ":""}` przechodzi walidację i renderuje puste
`image:`. **Potwierdzone na żywo (test #4).** **Zalecenie:** walidować po
normalizacji kluczy albo nie nadpisywać wartości zadeklarowanych zmiennych.

### F-03 · Średnia · Manualny update z wykluczeniami blokowany domyślnym `remove_orphans`
`internal/httpapi/handler.go:1050`. `RemoveOrphans` defaultuje do `true`, a
`RunUpdate` odrzuca kombinację z wykluczeniami. Każdy klient wysyłający
`excluded_services` bez jawnego `remove_orphans:false` dostaje 400.
**Potwierdzone na żywo (test #7).** **Zalecenie:** przy obecnych wykluczeniach
domyślnie `remove_orphans=false` (lub walidować z jasnym komunikatem po stronie UI).

### F-04 · Średnia · `${VAR}` szablonu koliduje z interpolacją Compose
`internal/stacks/templates.go:346`. Każdy niezadeklarowany `${WIELKIE}` w compose
szablonu → błąd „missing template variables". Wbudowane szablony są bezpieczne, ale
szablony użytkownika z częstym `${PUID}/${PGID}/${TZ}` (obrazy linuxserver.io)
nie dadzą się utworzyć. **Zalecenie:** podstawiać tylko zadeklarowane zmienne,
resztę `${...}` zostawiać nietkniętą dla runtime `.env`.

### F-05 · Średnia · `up`/`recreate` czyści ważny badge aktualizacji bez pull
`internal/httpapi/handler.go:2875` (+ odpowiednik w maintenancejobs). Zwykły
restart bez pobrania obrazu kasuje badge „update available" do następnego
sprawdzenia. **Zalecenie:** unieważniać badge tylko przy akcjach faktycznie
pobierających obraz (`pull`, `up --pull`).

### F-06 · Średnia · `last_deployed_at` znika, gdy compose staje się nieprawidłowy
`internal/stacks/service.go:884`. Przypisanie `LastDeployedAt` jest pod tym samym
guardem `ConfigState != invalid`, więc edycja compose do stanu nieprawidłowego
ukrywa znacznik ostatniego deployu, choć baseline nadal istnieje w DB.

### F-07 · Średnia · Wartości zmiennych bez escapowania (YAML injection)
`internal/stacks/templates.go:352`. Wartość jest tylko `TrimSpace`'owana i
wstawiana dosłownie — wewnętrzny znak nowej linii / cudzysłów psuje lub wstrzykuje
konfigurację. W modelu jednego użytkownika waga niższa, ale legalne wartości ze
znakami specjalnymi psują compose. **Zalecenie:** cytować/escapować wartości.
